2.1. The Concept of Data Visualization.
Data Visualization is regarded as “the science of visual representation of ‘data’” (Friendly, 2009, p. 2).
Traditional data visualisation focuses on communication for understanding data.
Advantages of traditional data visualization:
- Ability to represent a vast amount of data immediately
- Enables viewers to identify patterns immediately for forumalting ideas
- Can be used for product quality control, because immediate identification of problems is possible
- Enhances understanding of large-scale and small-scale data
- “data visualization assists in the constructing of hypotheses.”
Data visualization often results in graphical images which adssists in making decisions (Ware, 2012).

2.1.1. The Definition of Data Visualization.
“Data Visualization can be defined in many ways - most definitions focus on the connection between data and computer technology in order to transform data into a visual or sonic form.”
Card, Mackinlay, and Schneiderman define it as “the use of computer-supported, interactive, visual representation of data to amplify cognition.”
Manovich defines it as “a transformation of quantified data which is not visual into visual representation?
Friendly (2009) defines it as “information which has been abstracted in some schematic form, including attributes or varibales for the units of information”
Kirk states that “data visualization involves an information exhange that includes the messenger, the receiver, and the message” and defines data visualization as “the representation and presentation of data that exploits our visual perception abilities in order to amplify cognition”
Bikakis defines data visualization as “the presentation of data in a pictorial or graphical format, and a data visualization tool is the software that generates this presentation. Data visualization provides users with intuituve means to interactively explore and analyze daya, enabling them to effectively identify interesting patterns, infer correlations and causalities, and supports sense-making activities.”

Data visualization involves four aspects:
- Real-time interaction (Bikakis - real-time interaction requires effecient and scalable techniques)
- On-the-fly processing (the support of on-the-fly visualizations over large and dynamic sets of volatile raw data - required)
- Visual scalability
- User assistance and personalization.

A key feature of modern visualization is user comprehension and customization capabilies to various user-defined exploration scenarios and preferences.
Data vizualization can be categorized into two sub-fields: (Marai, 2010; Post et al., 2003)
- Information visualization - is used to visually represent abstract data such as business data (Card, Mackinlay, & Schneiderman, 1999; Spence, 2007)
- Scientific visualization - is used to represent scientific data (Spence, 2007)

Scientific and Information visualization focus on how to transform data into a visual form so that a user can read a graphical image and apprehend useful information.

2.1.2. The Definition of Data and Information
I like the concept of data not having “meaning” because it means that in visualizing it, we are giving it meaning ourselves - which can tie in with what Hanli said in the lecture about “meaning making”.
In the reading, data is defined as “unprocessed information”, which is interesting because information is defined as “processed data” in information science. Information can be considered “useful data”.

2.1.3. The Forms of Traditional Data Visualization
Forms of data representation are also called “visualization techniques”, which have been developed to optimize selecting and creating the most effective and efficient trandormation of data into information.
Visualization techniques are picked dependant on the type of data.
Data Visualization can be split into entities and relations (Ware, 2012) in which entities refer to “visualizing objects”, while relations refer to “structures and patterns relevant to those entities”.
Data visualization (information design) aims to present all the objective data required to enable the users to make a decision. (Wildbur & Burke, 1998)
This is another concept that interests me, as I don’t know how I feel about “objective data” being used in this sentence. Of course, the data is completely objective but presenting it usually is not - and in fact, research tends not to be objective either. For example, studies that suggest the colour BLUE is evil, may have been paid for by the colour RED. And that “objective” data maybe shared, while other objective data (perhaps implying that all colours are equally evil) is conveniently not displayed.
Statistics and data visualization cannot always be described as “objective.”
